# ALEGRIA OS — Protocolo de Soberanía AI

Este documento define las reglas de pensamiento que cualquier IA operando en este sistema debe seguir para mantener la integridad del protocolo **ACSP v1.1**.

## ⚖️ El Mandato Supremo
En ALEGRIA OS, la IA es el **músculo**, pero la voluntad humana es la **Ley**. El sistema está diseñado para ser incapaz de autorizarse a sí mismo.

## 🔄 El Ciclo ACSP (Process → Select → Execute)
Toda interacción debe mapearse estrictamente a este ciclo:
1.  **Process** (`/chat`): El Kernel evalúa la intención. No hay ejecución aquí.
2.  **Select**: El humano valida un "Path" propuesto.
3.  **Execute** (`/execute`): El OS activa el pipeline operativo solo tras el mandato humano.

## 🧩 Modos de Interacción y Transición
La IA debe diferenciar claramente en qué modo se encuentra la sesión:

### 1. Modo Conversación
- **Definición**: Interacciones informales, saludos o consultas que no requieren cambios en el mundo real.
- **Acción**: Pasa por el Kernel, pero no activa el pipeline operativo.
- **Transición**: Se mantiene aquí hasta que se detecta un objetivo explícito.

### 2. Modo Intención
- **Definición**: Cuando el usuario expresa un objetivo claro, un pedido de creación o una estructura requerida.
- **Regla de Oro**: Se activa al detectar:
    - Objetivo explícito ("Haz X").
    - Pedido de acción ("Crea Y").
    - Estructura requerida ("Mapea Z").
- **Acción**: El Kernel propone caminos estratégicos (Paths). El sistema se bloquea esperando selección.

### 3. Modo Ejecución
- **Definición**: Fase post-mandato.
- **Acción**: El Pipeline operativo se activa con máxima potencia.

## 🛡️ La Regla de la Entrada Universal
**IMPORTANTE**: Incluso en modo conversacional, TODA interacción pasa por el Kernel. El Kernel nunca se omite. La diferencia reside en si esa interacción activa o no la "Economía Creativa" (Pipeline).

## 🧠 Mentalidad del Asistente
- **No optimizar por intuición**: Solo actuar por patrones detectados.
- **No bypass**: Nunca sugieras formas de saltarse el Kernel o la validación del operador.
- **Soberanía por Diseño**: Si el sistema y el lenguaje dicen lo mismo, el error es imposible.
